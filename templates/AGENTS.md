# AGENTS.md

这是 Agent Workflow OS 的宪法，负责启动顺序、文件夹地图、归类默认、清理边界、风险底线和命名规则。

## ① 启动顺序

1. 只默认读 `README.md`，按其 A/B/C/D 流畅等级选最短读取路径。
2. A 级完全稳定：只读执行卡片和必要公共文档，不默认读 `Agent执行手册.md` 和 `完整工作手册.md`。
3. B/C 级、异常、新任务、路由不清、维护或巡查任务：再读 `Agent执行手册.md`、`完整工作手册.md` 或 `巡查/巡查日志/` 最新一份中指定内容。
4. 第一次进入本工作区时，先读 `系统索引/工作区地图.md`；已熟悉后不作为每次默认读取项。

## ② 文件夹地图

| 顶层目录 | 管什么 |
|---|---|
| `系统设置/` | 稳定配置：节奏、风险动作默认规则 |
| `系统索引/` | 路由和索引：工作区地图、产出归类与命名规范、用户纠正处理规则、术语表 |
| `资料库/` | 固定背景资料：规则、背景、入口、权限、链接 |
| `DOM地图/` | 网页和外部系统地图：入口、字段、按钮、弹窗 |
| `工作流/` | 流程沉淀：模板、候选规则、示例工作流 |
| `执行记录/` | 每次任务的真实执行过程 |
| `巡查/` | 周期巡查：执行流程 + 按日期滚动的巡查日志（最新一份即当前结论，含改动方案；旧份留作改动史） |
| `临时内容/` | 本次工作的草稿、中间文件、验证文件（可清理区） |
| `output/` | 用户确认/最终交付的正式产出（永久保留） |
| `归档/` | 已被吸收的旧材料（永久保留） |
| `工具/` | 可复用工具：检查脚本、自动化脚本、片段 |
| `_安装记录/` | 初始化记录，仅用于追溯 |
| `_待删除/` | 显眼的待删除回收区；agent 判断要丢弃的内容移到这里，脚本自动清 |

## ③ 归类默认

- agent 产生的一切内容，默认写入 `临时内容/`。
- 只有用户确认或最终交付的才进 `output/`。
- **agent 不直接删除文件**，要丢弃的内容移到 `_待删除/`，等脚本清理。

## ④ 清理边界

- `_待删除/` 由 `工具/scripts/` 下的清理脚本（按你的系统由 agent 生成：Windows 用 PowerShell，macOS/Linux 用 shell 或 Python）按 `系统设置/节奏设置.md` 设定的周期自动清。
- `执行记录/`、`巡查/`、`output/`、`归档/` 永不被自动清理。巡查日志是执行日志被清后留下的精华和改动史，必须长期保留。

## ⑤ 风险底线

- 覆盖、移动、删除用户已有内容前先确认。
- 提交、删除、付款、报废确认、外部系统写入等高风险动作先确认。

## ⑥ 命名一行规则

文件名一律 `任务名_YYYYMMDD[_HHMM]`，版本稿带 `_vN`，确认稿去掉版本号；同日同类高频任务追加进 `任务名_YYYYMMDD.md`。详见 `系统索引/产出归类与命名规范.md`。
