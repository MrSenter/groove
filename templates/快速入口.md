# 快速入口

本文件是唯一路由表。先用下表判断是否命中已知工作流，再按"流畅等级"决定读取范围。

| 用户说法 | 工作流 | 流畅等级 | 极速状态 | 本次读这些 |
|---|---|---|---|---|
|  |  |  |  |  |

> 还没有工作流。第一个真实任务做完后，按 `Agent执行手册.md` 的"新任务流程"沉淀候选流程，巡查确认后在此登记一行。
> 格式示例：`帮我处理客户工单` → `工作流/处理客户工单/workflow.md` → `C 已做过未稳定` → `workflow.md + candidates(按需) + 巡查/巡查日志/ 最新一份相关段落`。

## 规则

- 进入快速入口不等于完全稳定；必须看流畅等级决定读取范围。
- 同类任务出现 2 次，记为候选；出现 3 次，可生成候选工作流。
- 完全稳定 A 级不默认读巡查日志和完整手册。
- 没命中 → 读 `Agent执行手册.md` 的"新任务流程"。

## 极速模式（A 级之上的提速档，用户点头才升）

- 「极速状态」列取值：`—`（普通）/ `可逆·极速` / `不可逆·极速` / `已拒绝`；爬坡期记 `连续N/5`。
- **怎么升**：某工作流连续 5 次完整执行、不卡点 → **执行完第 5 次时当场在对话框问用户**要不要升极速。**用户没回答就当没看到，下次执行完再问**；只有明确选了才定：说要→升级，说不要→标 `已拒绝`、不再问。用户也可随时主动要求。
- **可逆·极速**（可重跑、对后续无影响）：可省核对，不卡点就不写执行日志。
- **不可逆·极速**（提交/删除/写线上等不可回滚）：只省读取/叙述，**数量等廉价校验和风险确认仍保留，且仍写一行回执**。
- **怎么降**：任何异常 / 用户纠正 / 数量等不符 → 当场退回普通、恢复日志、清零连续次数，并告知用户。
- 完整原则见 `完整工作手册.md`「极速模式」；连续次数由巡查维护。
