# 产出归类与命名规范

## 产出归类规则

| 类型 | 默认位置 | 说明 |
|---|---|---|
| 临时草稿 | `临时内容/草稿/` | 默认到期清理 |
| 中间文件 | `临时内容/中间文件/` | 默认到期清理 |
| 验证文件 | `临时内容/验证文件/` | 关联正式结果时可转入验证证据 |
| 待确认版本稿 v1/v2/v3 | `临时内容/草稿/` | Excel、PPT、文档、图片等多轮版本，未被用户确认前都不是正式产出 |
| 正式结果 | `output/正式产出/` | 长期保留 |
| 可复用样例 | `output/可复用样例/` | 复盘确认后长期保留 |
| 验证证据 | `output/验证证据/` | 与最终结果关联时保留 |
| 工作流经验 | `工作流/{名称}/candidates.md` | 验证后再进主流程 |
| 网页路径/字段 | `DOM地图/{系统名}.md` | 长期保留 |
| 固定业务规则 | `资料库/固定规则.md` | 长期保留 |
| 已吸收旧材料 | `归档/` | 巡查吸收后归档 |

## 版本稿处理规则

- `v1`、`v2`、`v3`、`draft`、`review`、`修改稿` 等未确认版本，默认放 `临时内容/草稿/`。
- 如果版本稿是脚本生成的中间 Excel、PPT、图片或转换文件，放 `临时内容/中间文件/`。
- 用户明确确认的最终稿，才移动或保存到 `output/正式产出/`。
- 与最终稿强相关、需要保留的对比截图或检查结果，放 `output/验证证据/`。
- 确认稿出现后，旧版本默认仍是待处理临时内容：彻底不要的移到 `_待删除/`；需要追溯才移入 `归档/旧产出/`，并写明原因。
- 不要把 v1/v2/v3 自动搬进 `资料库/`。只有从多轮版本中提炼出的稳定规则，才写入 `资料库/` 或对应工作流。

## 命名规范

### 原则

- 任务名在前、日期在后，同类自动聚在一起好查。
- 日期格式统一：`YYYYMMDD`，时间追加 `_HHMM`。
- 同日同类高频任务追加进 `任务名_YYYYMMDD.md`，不新建多个文件。
- 版本稿带 `_vN`，用户确认后去掉版本号变为确认稿。

### 命名规范表

| 区域 | 命名格式 | 例 |
|---|---|---|
| 执行记录·一次性/低频 | `{任务名}_{YYYYMMDD_HHMM}.md` | `客户工单处理_20260616_1500.md` |
| 执行记录·同日高频同类（追加） | `{任务名}_{YYYYMMDD}.md` | `订单批量处理_20260616.md` |
| 巡查日志（含改动方案，按日期滚动保留） | `巡查日志/巡查日志_{YYYYMMDD_HHMM}.md` | |
| DOM地图 | `DOM地图/{系统名}.md` | `DOM地图/某后台系统.md` |
| output 确认稿 | `{交付物}_{YYYYMMDD}.{ext}`（去版本号） | |
| 待删除/版本稿 | `{用途}_{YYYYMMDD_HHMM}_v{n}.{ext}` | |
| 归档 | 原名 + `_归档{YYYYMMDD}` | |
