# 术语表

| 术语 | 含义 |
|---|---|
| 原工作流评分 | 执行前对既有 `workflow.md` 的可执行性评分 |
| 成功读回 | 每一步完成后能看到、生成或验证的成功证据 |
| 候选流程 | 新任务或未验证经验的初步流程，通常放在 `candidates.md` |
| DOM 地图 | 网页或外部系统的入口、字段、按钮、反馈和卡点地图 |
| 工具化 | 将稳定、低风险、高频步骤做成模板、检查器、脚本或半自动工具 |
| 流畅等级（A/B/C/D） | 本次执行要读多少文档：A=只读执行卡片；B=+候选/样例；C=完整流程+巡审日志；D=新任务流程 |
| 成熟度（L0–L5） | 工作流的长期建设阶段，从 L0（无文档）到 L5（全自动化） |
| 巡查 | 把执行记录压缩成更短入口、规则、流程等级和工具化建议的周期动作 |
