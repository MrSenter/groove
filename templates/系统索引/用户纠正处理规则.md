# 用户纠正处理规则

用户纠正先进入候选区，巡查确认后再进入主工作流。

| 用户说法 | 判断 | 处理 |
|---|---|---|
| 不是这样 | 本次理解偏差 | 写执行记录 |
| 以后都这样 | 长期规则候选 | 写 `candidates.md` 或 `资料库/固定规则.md` |
| 下次直接按这个 | 强复用信号 | 提升候选工作流优先级 |
| 这个文件留着 | 长期保存信号 | 移入正式产出或样例 |
| 这个不用留 | 临时内容信号 | 保持临时, 按周期清理 |
| 页面入口变了 | DOM 更新 | 更新 `DOM地图/{系统名}.md` |

标准转译:

```text
当什么情况出现时,
判断依据是什么,
正确动作是什么,
成功标准是什么,
失败时怎么处理。
```
