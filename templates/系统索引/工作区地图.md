# 工作区地图

本文件是初步地图，说明这个工作区每个区域负责什么。第一次进入本工作区时先读本文件；熟悉后按 `README.md` 分流即可。

## 入口区

| 位置 | 用途 | 日常是否读取 |
|---|---|---|
| `AGENTS.md` | 硬入口和安全底线（宪法） | 自动入口 |
| `README.md` | 日常分流（路由壳，约 10 行） | 是 |
| `快速入口.md` | 唯一路由表（工作流 + 流畅等级 + 读取范围） | 是 |
| `Agent执行手册.md` | 异常、新任务、B/C 级流程和维护任务的执行纪律 | 按流畅等级读取 |
| `完整工作手册.md` | 完整系统规则 | 新任务、维护或巡查时读取 |

## 工作区间

| 区域 | 用途 | 典型内容 |
|---|---|---|
| `系统设置/` | 稳定配置 | 节奏设置、风险动作默认规则 |
| `系统索引/` | 路由和索引 | 工作区地图、产出归类与命名规范、工作流总览、用户纠正处理规则、术语表 |
| `资料库/` | 固定背景资料 | 固定规则、业务背景、网站入口（常用链接地图）、权限说明 |
| `DOM地图/` | 网页和外部系统地图 | 页面入口、字段、按钮、弹窗、滚动层、成功读回、常见失败 |
| `工作流/` | 流程沉淀 | 候选流程规则、候选流程、成熟流程；成熟流程入口为 `workflow.md` |
| `执行记录/原始记录/` | 原始执行记录 | 每次任务的真实过程记录 |
| `巡查/巡查日志/` | 按日期滚动的巡查日志 | 每份含发现的问题、已固化/未定的工作流、本次改动方案、执行统计；最新一份即当前记忆，旧份留作改动史；B/C 级、异常或卡顿时读最新一份相关段落 |
| `临时内容/` | 本次工作的草稿/中间/验证区 | 草稿、中间文件、验证文件（可清理） |
| `output/` | 长期保留内容 | 正式产出、可复用样例、验证证据 |
| `工具/` | 可复用工具 | 检查脚本、自动化脚本、片段 |
| `归档/` | 已吸收旧材料 | 旧工作流、旧日志、旧产出 |
| `_安装记录/` | 初始化记录 | 用于追溯初始化过程 |
| `_待删除/` | 显眼的待删除回收区 | agent 判断要丢弃的内容移到这里；脚本按节奏设置自动清 |

## 默认判断

- 任务路由不清：先看 `README.md`，再看 `快速入口.md`。
- 命中已知工作流：按快速入口的流畅等级分流，读最少文件。
- 快速入口暂无对应项：读 `Agent执行手册.md` 的"新任务流程"。
- B/C 级或异常：读 `巡查/巡查日志/` 最新一份的相关段落，不翻旧原始记录。
- 文件归类不清：看 `系统索引/产出归类与命名规范.md`。
- 风险动作判断不清：看 `系统设置/风险动作默认规则.md`。
- 发现重复任务：先记执行记录或候选经验，巡查确认后再升级为工作流。
- 探索网站或外部系统：同步补 `DOM地图/{系统名}.md`，让下个 agent 复用入口、字段和稳定定位方式。
- 要丢弃文件：移到 `_待删除/`，不要直接删除。
