# 完整工作手册

本文件保存低频系统原则。日常执行先读 `README.md`，按流畅等级选择最短读取路径；新建/调整工作流、巡查、维护规则或路由不清时，才进入本文件。

## 入口分层

- `AGENTS.md`: 硬入口，负责启动底线。
- `README.md`: 日常入口，包含快速入口、流畅等级和最短读取路径。
- `Agent执行手册.md`: 异常、新任务、B/C 级流程、评分、日志和收尾检查时读取。
- `系统索引/工作区地图.md`: 初步地图，说明每个区域负责什么。
- `完整工作手册.md`: 新任务、维护、巡查和路由不清时使用的完整规则。
- `资料库/`: 固定规则、背景、网站入口（常用链接地图）和权限。
- `DOM地图/`: 网站和外部系统页面地图。
- `巡查/`: 按日期滚动的巡查日志（最新一份即当前结论，含改动方案；旧份留作改动史）。
- `系统设置/`: 节奏和风险动作设置。
- `系统索引/`: 候选路由、产出归类、纠正处理等维护索引。
- `工作流/`: 具体可复用工作流。

## 权威规则位置

- 日常分流和 A/B/C/D 流畅等级：`README.md`
- 执行纪律、新任务流程、评分和收尾：`Agent执行手册.md`
- 产出、临时、归档、多版本文件边界：`系统索引/产出归类与命名规范.md`
- 巡查具体执行步骤：`巡查/巡查执行流程.md`
- 最新未决事项和下次注意点：`巡查/巡查日志/` 最新一份

## 周期巡查

巡查目标是把执行记录中的稳定经验压缩成更短的入口、规则、流程等级和工具化建议。具体步骤以 `巡查/巡查执行流程.md` 为准。

## 极速模式（可逆性分档的提速档）

极速模式是 A 级稳定工作流之上的提速档，不是新等级，也不替代风险边界。它只减少**读取、叙述、探索性核对**的成本，按任务**可逆性**决定能省到哪一步。

- **可逆·极速**：任务可重跑、对后续无影响（纯生成、只读导出之类）。可省掉额外核对；只要不出异常就不写执行日志（出异常那一单仍写，并据此降级）。
- **不可逆·极速**：任务不可回滚（提交、删除、付款、覆盖、写入线上）。可省读取和叙述，但**廉价不变量校验（数量 / 选中数 / 成功回读）和风险确认必须保留**，且每次仍写至少一行回执——它要喂巡查统计、留漂移痕迹。

升降级握手：

- **升**：工作流连续 5 次完整执行不卡点 → **执行完第 5 次时当场在对话框问用户**。**没回答视为没看到，下次执行完再问**；只有明确选择才定下来——确认才升，拒绝则标 `已拒绝`、不再问。用户也可主动要求（让不可逆任务进极速，等于用户预授权减少检查）。
- **降**：任何异常 / 用户纠正 / 不变量不符 → 当场退回普通模式、恢复完整日志、连续次数清零，并告知用户。
- **计数**：每个工作流的"连续不卡点次数"由巡查在 `巡查/巡查日志/` 维护。爬坡期（仍写回执）才计数；可逆·极速升档后不再计数，靠出异常自曝降级；不可逆·极速一直写回执，巡查持续盯连续次数和漂移。

## 工具化策略

优先工具化高频、低风险、输入输出清楚、失败容易回滚的步骤。涉及提交、删除、付款、报废确认、外部系统不可逆写入的流程，先做检查清单、草稿生成、只读验证或半自动工具。

工具化等级：

| 等级 | 含义 |
|---|---|
| T0 | 无工具 |
| T1 | 模板 / 检查清单 |
| T2 | 检查脚本 |
| T3 | 草稿生成器 |
| T4 | 半自动执行 |
| T5 | 自动执行但需要确认 |
| T6 | 全自动执行 |

## 候选流程目标

候选流程不是泛泛总结，目标是减少下一次摸索成本。能写成脚本的任务，要记录到接近脚本参数表和断言表的精度。

候选流程至少包含：

- 触发说法：用户通常会怎么提出这个任务。
- 必要输入：字段名、文件、账号、页面、日期、数量等。
- 稳定入口：从哪里进入任务页面或处理目录。
- 标准动作：每一步做什么。
- 成功读回：每一步完成后看哪个字段、日志、页面状态、文件结果。
- 失败分支：找不到入口、字段变化、数量不一致、权限不足、提交失败时怎么判断。
- 地图更新：如果探索了网站或外部系统，补到对应 `DOM地图/{系统名}.md`。
- 工具化线索：哪些步骤可变成脚本、脚本需要哪些参数、需要哪些检查。

## 用户只需做什么

- 正常提出任务。
- 指出 AI 做错的地方。
- 对高风险动作做确认。
- 在巡查时确认少量候选规则。

## 工作方式

- 用户提出当前任务，agent 从真实任务中学习重复模式。
- 安装说明归档在 `_安装记录/`，用于追溯初始化过程。
- 已有工作流保持稳定；新经验先进入候选区。
- 一次错误先写入执行记录；稳定纠正经过巡查后进入工作流。
- 探索过的页面和外部系统信息沉淀为地图，供下个 agent 直接复用。

## 进入本手册的场景

- 用户要求修改工作平台、工作流、规则或目录结构。
- `README.md` 的快速入口暂无对应项。
- 巡查执行记录、清理临时内容、升级候选规则。
- agent 对产物归类、风险动作或工作流成熟度没有把握。
- 需要调整越用越省 token 的入口、流畅等级或巡查压缩规则。
